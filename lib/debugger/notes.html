Debugger Release Notes

Debugger 2.0

Fixed errors and malfunctions

  • Parts of the code, mainly for the GUI, has been rewritten in order to make Debugger more robust and efficient. The functionality remains the same but behaviour and appearance have changed slightly. Modules must have been compiled with the option debug_info set in order to be interpreted, only modules for which this is true are shown in the Interpret window. (Own Id: OTP-4201)
  • The "Interpret Dialog" window no longer hangs if one tries to debug-compile a module residing in a sticky directory.
    (Own Id: OTP-2665)

There are also release notes for older versions.
